### 2.2 コンポーネント

#### 検索バー（SearchBar）

- プレースホルダー: "Search players..."
- 入力後 300ms debounce で API リクエスト送信
- 検索中はローディングスピナーを表示
- 入力クリアボタン（×）

#### プレイヤーカード（PlayerCard）

各カードに表示する情報：

| 項目 | 表示内容 |
|---|---|
| ハンドルネーム | テキスト（太字） |
| 国旗 | 国籍に応じた絵文字フラグ |
| ロール | バッジ（Tank / DPS / Support / Flex） |
| 現所属チーム | チーム名テキスト（なければ "Free Agent"） |

クリックで S-02 プレイヤープロフィールページへ遷移。

#### 検索結果なし

```
No players found for "{query}"
```

### 3.2 セクション詳細

#### 基本情報ヘッダー

| 項目 | 表示条件 |
|---|---|
| ハンドルネーム | 常に表示 |
| 本名 | データがある場合のみ |
| ロール | データがある場合のみ（バッジ形式） |
| 国籍 | データがある場合のみ（国旗絵文字 + 国名） |
| 生年月日 | データがある場合のみ |

#### 現所属チーム

- `isActive = true` の TeamRoster が存在すれば表示
- なければ "Free Agent" を表示

#### キャリア履歴（Career History）

- TeamRoster を `joinDate DESC` で並べる
- `leaveDate = null` の場合は "Present" と表示
- テーブル形式: チーム名 / 在籍期間

#### 移籍履歴（Transfer History）

- Transfer レコードを `date DESC` で並べる
- `fromTeam = null` → "Free Agent → {チーム名}"
- `toTeam = null` → "{チーム名} → Free Agent"
- 通常: "{fromTeam} → {toTeam}"

#### Liquipedia リンク

- `liquipediaUrl` がある場合のみ表示
- 外部リンクとして別タブで開く

### 3.3 エラー・ローディング状態

| 状態 | 表示 |
|---|---|
| 読み込み中 | スケルトン UI |
| 選手が見つからない（404） | "Player not found" + 検索ページへ戻るリンク |

## 4. デザイン方針

### 4.1 カラーパレット

Overwatch のブランドカラーを参考にしつつ、ダークテーマをベースとする。

| 用途 | カラー |
|---|---|
| 背景 | `#0f1117`（ほぼ黒） |
| カード背景 | `#1a1d27` |
| プライマリ | `#f99e1a`（Overwatch オレンジ） |
| テキスト | `#e5e7eb` |
| サブテキスト | `#6b7280` |

### 4.2 ロールバッジ カラー

| ロール | カラー |
|---|---|
| Tank | `#4e9aff`（青） |
| DPS | `#ff4e4e`（赤） |
| Support | `#4eff91`（緑） |
| Flex | `#c084fc`（紫） |

### 4.3 レスポンシブ対応

- ブレークポイント: `sm (640px)` / `md (768px)` / `lg (1024px)`
- 検索結果カード: モバイルは1列、タブレット以上は2〜3列グリッド
- プロフィールページ: シングルカラム（全幅最大 `760px`）
