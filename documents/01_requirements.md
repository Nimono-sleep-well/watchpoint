# 要件定義書

**プロジェクト名**: Watchpoint  
**作成日**: 2026-05-26  
**ステータス**: MVP フェーズ

---

## 1. プロジェクト概要

### 1.1 目的

Overwatch の国内外プロシーン（OWCS・旧OWL・オープントーナメント・地域リーグ）を網羅した、選手・チーム・試合データベースサイトを構築する。vlr.gg（Valorant向けEsportsデータベース）に相当するサービスのOverwatch版として位置づける。

### 1.2 フェーズ定義

| フェーズ | 内容 |
|---|---|
| **MVP（本文書の対象）** | プレイヤー検索・プロフィール閲覧のみ |
| Phase 2 | 試合結果・スケジュール |
| Phase 3 | チームページ・大会ページ |
| Phase 4 | 統計分析（ヒーロー別スタッツ・マップ分析・メタ分析） |

### 1.3 OSSポリシー

- コードは GitHub で公開（ライセンス: MIT 予定）
- データソースの Liquipedia コンテンツは CC-BY-SA 3.0 に従い出典を明記
- セルフホスト可能な構成とする（docker-compose提供）

---

## 2. 対象ユーザー

| ユーザー | 説明 |
|---|---|
| **サイト訪問者** | Overwatch プロシーンのファン。選手情報を調べたい |
| **セルフホスト利用者** | OSSとして自分のサーバーで動かしたい開発者 |
| **サイト運営者** | 本番サーバーを管理し、Liquipedia APIキーを保有する |

---

## 3. 機能要件（MVP）

### 3.1 プレイヤー検索

| # | 要件 |
|---|---|
| F-01 | ゲーム内名（ハンドルネーム）でプレイヤーを検索できる |
| F-02 | 部分一致・前方一致で検索できる（大文字小文字を区別しない） |
| F-03 | 検索結果に名前・国籍・ロール・現所属チームを表示する |
| F-04 | 検索結果が0件のときはその旨を表示する |

### 3.2 プレイヤープロフィール

| # | 要件 |
|---|---|
| F-05 | 選手の基本情報（ハンドルネーム・本名・国籍・生年月日・ロール）を表示する |
| F-06 | 現在の所属チームを表示する |
| F-07 | チーム在籍歴（チーム名・加入日・離脱日）を時系列で表示する |
| F-08 | 各選手のプロフィールページは固定URL（`/players/[id]`）でアクセスできる |
| F-09 | Liquipedia の該当ページへのリンクを表示する |

---

## 4. 非機能要件

| # | 要件 |
|---|---|
| NF-01 | 検索レスポンスは1秒以内（自前DBへのクエリのみ、Liquipediaへのリアルタイムアクセスなし） |
| NF-02 | プレイヤーデータは Liquipedia から定期同期し、最大24時間以内の鮮度を保つ |
| NF-03 | モバイルブラウザで閲覧できるレスポンシブデザインとする |
| NF-04 | セルフホスト時は `.env` に APIキーを設定するだけで動作する構成とする |

---

## 5. 制約条件

### 5.1 Liquipedia API 制限

| 項目 | 内容 |
|---|---|
| レート制限 | 60リクエスト/時間（LiquipediaDB API、承認後） |
| 認証 | カスタム User-Agent ヘッダーが必須 |
| 無料利用条件 | OSSとして公開 + 非営利 |
| ライセンス | CC-BY-SA 3.0（出典明記必須） |
| 禁止事項 | Liquipedia の機能・UIの複製、ギャンブル関連 |

### 5.2 対応策

- サーバー側でデータをキャッシュ（自前DB）し、Liquipediaへのアクセスは同期バッチのみ
- 同期バッチは1時間あたり最大60リクエストに収まるよう設計する

## 6. 対象データ範囲

### 競技シーン（将来フェーズ含む）

- Overwatch Champions Series (OWCS)
- Overwatch League（2018〜2023、過去データ）
- オープントーナメント・草大会
- 地域リーグ（APAC / NA / EMEA / KR）

### MVP で取得するデータ

- プレイヤー基本情報
- チーム在籍歴（TeamRoster）
- 移籍記録（Transfer）
